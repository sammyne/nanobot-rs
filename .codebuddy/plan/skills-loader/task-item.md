# 实施计划：Rust 版 Skills Loader 组件

- [ ] 1. 创建 Crate 基础结构
   - 在 `crates/skills/` 目录创建新的 Workspace 成员 crate
   - 配置 `Cargo.toml`，添加 `serde`、`serde_yaml`、`serde_json`、`anyhow` 等依赖
   - 创建 `lib.rs` 入口文件
   - _需求：技术约束 3、4_

- [ ] 2. 定义核心数据结构
   - 定义 `SkillMeta` 结构体，包含 `description`、`always`、`requires`、`metadata` 字段
   - 定义 `Requires` 结构体，包含 `bins` 和 `env` 字段
   - 定义 `Skill` 结构体，包含名称、路径、元数据、可用状态
   - 为所有结构体实现 `Deserialize` trait
   - _需求：2.1、2.2、3.1_

- [ ] 3. 实现 YAML 前置数据解析
   - 实现函数从 Markdown 文件中提取 `---` 包围的 YAML 块
   - 实现 `SkillMeta` 的解析逻辑，处理解析失败返回空元数据
   - 实现 `metadata` 字段的 JSON 解析，支持 `nanobot` 和 `openclaw` 键名
   - 编写单元测试验证解析逻辑
   - _需求：2.1、2.2、2.3、2.4_

- [ ] 4. 实现依赖检查功能
   - 实现检查 CLI 工具是否在 PATH 中可用的函数
   - 实现检查环境变量是否已设置的函数
   - 实现 `Skill` 的 `is_available` 方法，综合检查所有依赖
   - 实现 `missing_dependencies` 方法，返回缺失依赖描述
   - 编写单元测试
   - _需求：3.2、3.3、3.5_

- [ ] 5. 实现技能发现与加载
   - 实现 `SkillsLoader` 结构体，持有工作空间路径和内置技能目录路径
   - 实现扫描目录查找 `SKILL.md` 文件的功能
   - 实现技能列表功能，支持工作空间优先同名技能去重
   - 实现加载单个技能内容的功能
   - 编写单元测试
   - _需求：1.1、1.2、1.3、1.4、1.5_

- [ ] 6. 实现技能摘要构建
   - 实现 `build_summary` 方法，生成 XML 格式的技能摘要
   - 包含技能名称、描述、路径、可用状态
   - 对不可用技能包含缺失依赖信息
   - 编写单元测试
   - _需求：4.1、4.2、4.3、4.4_

- [ ] 7. 实现上下文技能加载
   - 实现移除 YAML 前置数据、仅保留 Markdown 正文的功能
   - 实现批量加载技能内容的功能，使用分隔符格式化
   - 为每个技能添加名称标题
   - 编写单元测试
   - _需求：5.1、5.2、5.3_

- [ ] 8. 实现 Always 技能识别
   - 实现获取 always 技能列表的方法
   - 筛选 `always=true` 且满足依赖的技能
   - 编写单元测试
   - _需求：6.1、6.2_

- [ ] 9. 集成测试
   - 创建测试用的技能目录和示例 `SKILL.md` 文件
   - 测试完整的加载流程
   - 测试各种边界情况（空目录、无效 YAML、缺失依赖等）
   - _需求：全部_
