# 实施计划

- [ ] 1. 集成 wagoid/commitlint-github-action 到构建流程
   - 在 `.github/workflows/build.yml` 中添加提交消息校验步骤
   - 配置步骤仅在 pull_request 事件时执行
   - 将校验步骤置于格式检查之前，确保校验失败时阻止后续步骤
   - _需求：1.1、1.2、4.1、4.2、4.3、4.4、5.1、5.2_

- [ ] 2. 创建 commitlint 配置文件
   - 在项目根目录创建 `.commitlintrc.yml` 配置文件
   - 配置允许的标准类型：feat、fix、docs、style、refactor、test、build、ci、chore、revert
   - 配置规则以支持破坏性变更标记（`!`）
   - _需求：2.1、2.2、2.3、2.4_

- [ ] 3. 测试提交消息校验功能
   - 创建测试 PR，包含符合规范的提交消息，验证能够通过校验
   - 创建测试 PR，包含不符合规范的提交消息，验证能够正确拒绝并显示错误信息
   - 创建测试 PR，包含多个不符合规范的提交，验证能够列出所有错误
   - 创建空提交的 PR，验证能够正确处理
   - _需求：1.2、2.5、3.1、3.2、3.3_

- [ ] 4. 验证手动触发场景
   - 通过 workflow_dispatch 手动触发构建，验证非 PR 上下文时跳过校验
   - 确认手动触发时其他构建步骤正常执行
   - _需求：5.1、5.2_
