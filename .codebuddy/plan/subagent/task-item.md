# Subagent 组件实施计划

- [ ] 1. 创建核心数据结构和类型定义
   - 定义 SubagentManager 结构体及其必需字段（LLM 提供商、工作区路径、消息总线发送端）
   - 定义 Task 结构体，包含任务 ID、描述、标签、通道、聊天 ID 等字段
   - 定义 TaskResult 枚举，区分成功和失败结果
   - 定义运行时状态结构（运行任务集合、任务计数器等）
   - _需求：1、6_

- [ ] 2. 实现 SubagentManager 初始化和配置
   - 实现 SubagentManager::new() 方法，接收必需和可选参数
   - 实现模型名称、温度、最大令牌数的默认值处理逻辑
   - 实现工具执行配置的默认值处理逻辑
   - 添加配置验证逻辑
   - _需求：6_

- [ ] 3. 实现子代理创建和管理功能
   - 实现 create_task() 方法，生成唯一任务 ID 和标签
   - 实现异步任务启动逻辑，使用 tokio::spawn
   - 实现运行任务集合的维护（添加和移除）
   - 实现 get_running_count() 方法，返回当前运行中的子代理数量
   - _需求：1_

- [ ] 4. 实现系统提示构建器
   - 创建 SystemPromptBuilder 结构体
   - 实现时间戳和时区信息的添加逻辑
   - 实现角色和目的说明的添加逻辑
   - 实现规则列表的添加逻辑（保持专注、只完成任务等）
   - 实现允许和禁止操作列表的添加逻辑
   - 实现路径信息和总结要求的添加逻辑
   - 实现 build() 方法，生成最终系统提示字符串
   - _需求：4_

- [ ] 5. 实现工具注册和执行模块
   - 创建工具注册表结构，管理可用工具
   - 实现文件系统工具（读取、写入、编辑、列出目录）的注册
   - 实现 Shell 执行工具的注册
   - 实现工具参数验证逻辑
   - 实现工具执行逻辑和结果返回
   - 实现未注册工具的拒绝逻辑
   - _需求：2_

- [ ] 6. 实现子代理执行循环
   - 实现 execute_task() 异步方法，处理单个任务执行
   - 实现消息上下文初始化逻辑（系统提示 + 任务描述）
   - 实现 LLM 调用循环，包含最大迭代次数限制（默认 15 次）
   - 实现工具调用解析和批量执行逻辑
   - 实现工具结果添加到消息上下文的逻辑
   - 实现无工具调用时的终止逻辑
   - 实现错误捕获和结果转换逻辑
   - _需求：3、7_

- [ ] 7. 实现结果通知和消息总线集成
   - 实现 InboundMessage 的封装逻辑
   - 实现成功结果的格式化（标签、任务描述、结果）
   - 实现失败结果的格式化（错误信息）
   - 实现结果消息的元数据设置（source: "system", sender_id: "subagent"）
   - 实现通过消息总线发送入站消息的逻辑
   - 实现原始通道和聊天 ID 的保持逻辑
   - _需求：5_

- [ ] 8. 实现日志记录功能
   - 为子代理创建添加 info 级别日志（任务 ID、标签）
   - 为任务开始执行添加 info 级别日志（任务 ID、描述）
   - 为工具执行添加 debug 级别日志（任务 ID、工具名称、参数）
   - 为任务成功完成添加 info 级别日志（任务 ID）
   - 为任务失败添加 error 级别日志（任务 ID、错误详情）
   - 为结果发布添加 debug 级别日志（任务 ID、目标通道、聊天 ID）
   - _需求：8_

- [ ] 9. 实现错误处理机制
   - 在关键执行路径添加 try-catch 错误捕获
   - 实现错误日志记录，包含任务 ID 和详细错误信息
   - 实现错误信息转换为 TaskResult 的逻辑
   - 确保错误结果通过消息总线正确通知
   - 实现错误状态标识（"error"）的设置
   - _需求：7_

- [ ] 10. 编写测试用例
   - 测试子代理创建、执行和完成的基本流程
   - 测试文件系统和 Shell 工具的执行和结果处理
   - 测试异常场景（工具失败、LLM 超时、参数无效等）
   - 测试不同配置组合（模型名称、温度、工具超时等）
   - 将测试代码组织到独立的 tests.rs 文件
   - 使用描述性函数命名（不使用 test_ 前缀）
   - _需求：10_
