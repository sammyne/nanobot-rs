# 需求

## 目标与背景

HeartbeatService::decide 方法在解析 LLM 返回的工具参数时可能失败，导致无法获得明确的决策结果。当前实现直接返回错误，中断心跳检查流程。

本需求旨在通过重试机制和错误反馈，让 LLM 有机会自我修正输出格式，提高心跳决策的成功率和稳定性。

## 功能需求列表

### 核心功能

- **重试机制**：当工具参数解析失败时，将错误信息反馈给 LLM 并允许其重新尝试
- **最大重试次数限制**：防止无限重试，设置最大重试次数为 1 次
- **错误消息格式化**：生成友好的错误提示，告知 LLM 期望的参数格式
- **优雅降级**：重试耗尽后仍失败时，降级为 Skip 决策，避免阻塞心跳循环

### 辅助功能

- **详细日志记录**：在解析失败时记录日志，包括尝试次数、错误原因
- **保留对话上下文**：将 LLM 的原始回复内容也加入对话上下文，帮助 LLM 理解上下文

## 非功能需求

- **性能**：重试机制不应显著增加心跳延迟
- **安全**：错误消息不暴露原始参数细节，避免敏感信息泄露
- **兼容性**：保持现有 API 不变，对调用者透明
- **可维护性**：代码结构清晰，易于调试和扩展

## 边界与不做事项

- 不实现无限重试或可配置重试次数（当前固定为 1 次）
- 不修改 Action 枚举的 serde 行为
- 不改变 Provider trait 接口
- 不在心跳主循环层面添加重试，只在 decide 方法的决策阶段重试

## 假设与约束

- **技术假设**：基于现有的 Message::tool 机制传递错误反馈
- **资源约束**：每次重试产生一次额外的 LLM API 调用
- **环境约束**：依赖 nanobot_provider::Message 类型和 ToolCall::parse_arguments 方法

## Action 参数格式

Action 是一个 untagged enum（默认 serde 行为），期望的 JSON 格式为：

```json
// Skip 变体（序列化为一字符串）
"skip"

// Run 变体（嵌套对象）
{"run": {"tasks": "需要执行的任务描述"}}
```

## 解析失败时的错误反馈消息

当解析失败时，反馈给 LLM 的错误消息格式如下：

```
Invalid arguments format. Error: <具体解析错误>.
Expected JSON format:
  - To skip: "skip"
  - To run tasks: {"run": {"tasks": "<task description>"}}
Please retry with a valid format.
```

消息设计原则：
- 不暴露原始 `arguments` 内容，保护安全性
- 明确给出正确格式示例，便于 LLM 修正
- 提示 "Please retry" 明确期望 LLM 重新输出
